# Understanding the Spider Graph

**Who this is for:** Freelancers

## What you'll achieve

- Learn how to read and interpret your spider (radar) graph
- Understand how skill points translate into visual proficiency levels

## Prerequisites

- Skills must be added to your profile
- Each skill must have points assigned
- At least one skill is required for meaningful graph visualization

## What the Spider Graph Represents

The spider graph is a **visual map of your skill points across multiple competencies**.

- Each axis = one skill (e.g., Frontend, Backend, APIs, Databases)
- Point distance from center = your proficiency in that skill
- Overall shape = balance or specialization of your skill set

**In simple terms:** More points -> farther from the center -> higher proficiency.

## How the Spider Graph Works

### Skill axes

*Expected result:* You can identify what each line represents

- Each spoke corresponds to a specific skill
- Skill names are displayed around the graph

### Point distance from the center

*Expected result:* Gauge your proficiency at a glance

- Closer to center = lower skill level
- Farther out = higher skill level

### Proficiency levels

*Expected result:* Understand colors or levels

- Levels may include: Learning, Beginner, Intermediate, Advanced, Expert
- Each level is visually differentiated for easy comparison

### Overall shape of the graph

*Expected result:* Interpret balance vs specialization

- Rounded shape -> broad skill coverage
- Spiked shape -> specialization in certain skills

### Total skills and expert level count

*Expected result:* Understand summary indicators

- Total number of skills shown below the graph
- Expert level count = how many skills reached the highest proficiency

### How Skill Points Affect the Graph

- Points control how far each node extends
- Increasing points moves the node outward
- Lowering points pulls it closer to the center
- The graph always reflects current skill points in real time

## FAQ

**What is a spider graph?**
A radar chart visualizing multiple skill values in a circular layout.

**Does the graph update automatically?**
Yes, it updates whenever skill points change.

**Can I have many low-level skills instead of few high ones?**
Yes, the graph shows either breadth or depth depending on point distribution.

**What does a small or empty graph mean?**
Few skills added or low points assigned.

**Do colors affect scoring?**
No. Colors only show proficiency levels; scoring depends on points.

**Is a balanced graph better than a specialized one?**
Not always. Balance suits generalist roles; spikes show niche expertise.

## Troubleshooting

- **Graph looks empty or very small** -> Add more skills or increase points
- **Skill missing** -> Ensure it is added and saved correctly
- **Graph not updating** -> Refresh page and confirm changes
- **Proficiency colors don't match expectations** -> Check point thresholds for each level
