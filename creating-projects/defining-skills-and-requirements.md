# Defining roles and required skills

**Who this is for:** Client

### What you'll achieve:

- Accurately tag your project to trigger high-quality Freelancer matches.
- Set realistic expectations for the expertise required to complete your project.

### Prerequisites:

- An active project draft.

### Steps:

- **Determine Seat Count:** Decide if you need a single specialist or a multi-person team.
- **Identify "Must-Have" Skills:** Select up to 5 core technologies required for the project. **Expected Result:** Freelancer with these verified skills will see your project first.
- **Add "Nice-to-Have" Skills:** List secondary skills that would be beneficial but aren't deal-breakers.
- **Set Seniority Levels:** Choose between Beginner, Intermediate, Advanced or Expert.
- **Note:** High-complexity tasks with low experience level often result in poor application quality.

### FAQ:

- **How many skills should I list?** We recommend 3-5 core skills. Listing 10+ can overwhelm applicants and lower your match quality.
- **Does "Seniority" impact my price?** Yes, businesses will want to pay higher rates for "Expert" level tags.
- **Can I require specific certifications?** Yes, you can mention these in the custom "Requirements" field.
- **What are "Seat Counts"?** The number of individual contracts you intend to open for this single project post.

### Troubleshooting:

- **"No matches found":** Your "Must-Have" list might be too specific (e.g., requiring 10 years of a 2-year-old framework). Try making some skills "Nice-to-Have."
